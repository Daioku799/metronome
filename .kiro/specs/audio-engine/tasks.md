# Implementation Plan

## 1. 基礎基盤と共有型定義
メトロノームエンジンの基盤となる通信型とプロジェクト構造を準備します。

- [x] 1.1 共有型定義と通信プロトコルの作成
  - メインスレッドとWorker間でやり取りするメッセージ（START, STOP, CONFIG, TICK）の型を定義する
  - 拍子のアクセントレベル（'strong', 'medium', 'weak', 'none'）の型を作成する
  - 型定義ファイルが作成され、他のコンポーネントから参照可能になる
  - _Requirements: 1.1, 3.1, 4.1, 4.2, 4.3, 4.4_

- [x] 1.2 Web Worker実行環境の構築
  - ViteなどのビルドツールでWorkerを読み込むための初期設定を行う
  - Workerのエントリポイントファイルを作成し、生存確認のログを出力する
  - メインスレッドからWorkerをインスタンス化し、通信ができることを確認する
  - _Requirements: 1.3_

## 2. オーディオエンジン・コア (Worker)
音生成とタイミング制御のロジックをWorkerスレッド内に実装します。

- [x] 2.1 (P) オーディオ・シンセサイザーの実装
  - Web Audio APIの `OscillatorNode` を用いたクリック音生成機能を実装する
  - アクセントレベルに応じた周波数（例: strong=880Hz, medium=660Hz, weak=440Hz）の切り替えを実装する
  - 'none' アクセントの場合は音を鳴らさない（ゲインを0にする、またはノードを接続しない）制御を実装する
  - `GainNode` によるマスターボリューム制御を実装する
  - 指定された時刻（AudioContext.currentTime）に正確に音を鳴らす関数が完成する
  - _Requirements: 1.1, 3.1, 3.2, 3.3_
  - _Boundary: src/audio/AudioSynthesizer.ts_


- [x] 2.2 (P) 拍シーケンス・スケジューラーの実装
  - 提供されたアクセントレベルのシーケンス（例: ['strong', 'weak', 'medium', 'weak']）を元に、次の拍のアクセントを決定するロジックを実装する
  - テンポ（BPM）に基づき、次の拍が鳴るべき秒数を計算する
  - 再起動時にシーケンスの先頭に戻るリセット機能を実装する
  - テスト入力に対して正しい拍の種類と間隔を計算できる
  - _Requirements: 2.1, 2.2_
  - _Boundary: BeatScheduler_

- [x] 2.3 スケジューリングループの実装
  - 25ms間隔の `setInterval` ループをWorker内に構築する
  - 「Look-ahead」方式により, 現在時刻から一定時間先までの拍を `AudioSynthesizer` にスケジュールする
  - 拍が発音されるタイミングでメインスレッドに `TICK` メッセージを送信する
  - 負荷がかかっても一定のテンポでクリック音が鳴り続ける
  - _Requirements: 1.2, 1.3, 4.3_

## 3. メインスレッド・インターフェース
UI層からオーディオエンジンを操作するための制御クラスを実装します。

- [ ] 3.1 オーディオエンジン・コントローラーの実装
  - Workerとの通信を隠蔽する `AudioEngineController` クラスを実装する
  - `start()`, `stop()`, `setTempo()`, `setMeter()` 等のメソッドを提供する
  - `setMeter` はアクセントレベルの配列（string[]）を受け取り, Workerへ転送する
  - Workerからの `TICK` メッセージを購読し、コールバックを実行する機能を実装する
  - 外部からメソッドを呼び出すことで、エンジンの状態を制御できる
  - _Requirements: 2.3, 4.1, 4.2, 4.4_

## 4. 統合と最終検証
システム全体を結合し、要件を満たしているか確認します。

- [ ] 4.1 全体統合テストと混合拍子の検証
  - コントローラーを介して ['strong', 'weak', 'medium', 'none'] のような複雑なアクセントシーケンスを再生し、正しいリズムと音色で音が鳴ることを確認する
  - テンポ変更がリアルタイムに反映されることを確認する
  - ブラウザの負荷をかけた状態でリズムの崩れがないか聴感でテストする
  - アクセントパターンが正しくループ再生される
  - _Requirements: 1.1, 2.1, 2.3, 4.1_

- [ ] 4.2* 単体テストによるロジックの網羅
  - `BeatScheduler` のエッジケース（空の配列、極端なテンポ等）の単体テストを追加する
  - `AudioEngineController` のメッセージ送信順序のテストを追加する
  - 全ての要件に対する単体テストがパスする
  - _Requirements: 2.1, 2.2, 4.1_
